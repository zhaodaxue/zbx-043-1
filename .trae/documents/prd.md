## 1. 产品概述

公交公司早高峰客流分析看板，用于可视化分析早高峰（6-9点）各站点的上下客不均衡情况，帮助运营人员识别客流压力站点和超载风险。

- 核心目标：通过多维度图表直观展示各站点净客流（上车-下车）分布和时段走势
- 目标用户：公交运营管理人员、调度人员、数据分析人员
- 产品价值：快速识别高客流压力站点，为运力调配和线路优化提供数据支撑

## 2. 核心功能

### 2.1 功能模块

1. **横向条形图**：各站点净客流（netFlow = 上车人数 - 下车人数）排名，按 netFlow 降序排列
2. **折线图**：展示选定站点在 6-9 点的 netFlow 时段走势
3. **超载风险表格**：列出 netFlow ≥ 80 的站点，标注早高峰超载风险等级
4. **站点交互**：点击条形图站点可联动折线图展示该站点时段数据

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 看板首页 | 顶部标题区 | 展示看板标题、数据更新时间、统计时段说明 |
| 看板首页 | 横向条形图 | 各站点净客流排名，支持点击选中站点，降序排列 |
| 看板首页 | 折线图 | 展示选中站点的 6-9 点 netFlow 走势曲线 |
| 看板首页 | 超载风险表格 | 展示 netFlow ≥ 80 的站点列表，含风险等级标注 |

## 3. 核心流程

用户进入看板页面 → 系统加载并解析 CSV 数据 → 计算各站点 netFlow 及聚合数据 → 渲染横向条形图和超载风险表格 → 用户点击条形图某站点 → 折线图联动更新展示该站点时段走势


## 4. 用户界面设计

### 4.1 设计风格

- **设计基调**：深色科技风，专业数据看板风格
- **主色调**：深蓝色系（#0f172a 背景，#1e293b 卡片），体现数据专业性
- **强调色**：青绿色（#06b6d4）用于正向数据，橙红色（#f97316）用于风险警示
- **字体**：使用现代无衬线字体，数字使用等宽字体提升可读性
- **布局**：卡片式栅格布局，左侧大尺寸条形图，右上折线图，右下风险表格
- **图标**：使用 lucide-react 线性图标，风格简洁统一

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|-----------|-------------|-------------|
| 看板首页 | 顶部标题区 | 大标题、副标题、时段标签、数据来源说明 |
| 看板首页 | 横向条形图 | 站点名称标签、净客流数值、进度条、悬停高亮、点击选中态 |
| 看板首页 | 折线图 | 时间轴 X 轴、netFlow Y 轴、折线曲线、数据点标记、站点名称标题 |
| 看板首页 | 超载风险表格 | 表头、站点名列、netFlow 数值列、风险等级标签、排序功能 |

### 4.3 响应式

桌面端优先设计，支持 1280px 以上屏幕最佳展示。中等屏幕自动调整为上下布局，小屏幕单列滚动展示。

### 4.4 动效与交互

- 页面加载时卡片依次淡入（staggered animation）
- 条形图进度条从 0 增长至目标值的动画
- 折线图路径绘制动画
- 站点悬停时高亮效果，点击选中状态变化
- 数据切换时平滑过渡动画
